# 初回実装 要求内容

## 変更履歴
- 2026-02-25: 初版作成

---

## 1. 概要

桃鉄ルールを踏襲した双六ゲームのWebブラウザ版を、段階的なフェーズに分けてゼロから実装する。

---

## 2. 実装スコープ

### Phase 0: 環境セットアップ
- Vite + TypeScript + Phaser.js の開発環境構築

### Phase 1: マップ・移動システム
- 日本全国マップの描画（都市・路線・マス目）
- プレイヤーコマの表示と移動
- サイコロ機能（演出含む）
- 分岐点での進路選択

### Phase 2: 物件・資産システム
- 物件の購入・保有・収益計算
- 独占ボーナス・年末決算処理

### Phase 3: カードシステム
- カードの取得・保持・使用・各効果実装

### Phase 4: 目的地・ボンビーシステム
- 目的地の設定と到着ボーナス
- ボンビーの憑依・行動・段階管理

### Phase 5: イベントシステム
- 月替わりランダムイベント

### Phase 6: CPU（AI）システム
- やさしい・ふつう・むずかしいの3段階実装

### Phase 7: セーブ・ロードシステム
- ローカルストレージへの保存・読み込み

### Phase 8: 演出・UI仕上げ
- アニメーション実装
- BGM・効果音の組み込み
- 全画面のUI完成

### Phase 9: テスト・品質チェック
- ユニットテスト・動作確認・コードレビュー

---

## 3. 受け入れ条件（必須）

- [ ] プレイヤー設定画面でプレイヤー数・CPU難易度・年数を設定できる
- [ ] 日本全国マップ上でコマを移動できる
- [ ] 物件を購入し、年末に収益を得られる
- [ ] カードを取得・使用できる
- [ ] 目的地に到着してボーナスを得られる
- [ ] ボンビーが最下位プレイヤーに憑依して妨害する
- [ ] 月替わりイベントが発生する
- [ ] CPUが3段階の難易度で自動行動する
- [ ] ゲームを中断・再開できる（ローカルストレージ保存）
- [ ] 設定年数終了後に最終ランキングが表示される

---

## 4. 制約事項

- 著作権のあるアセット（桃鉄の画像・音楽）は使用しない
- スマートフォン対応は行わない（PC専用）
- オンライン対戦は実装しない
